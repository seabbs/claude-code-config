Verify completed work against original requirements: $ARGUMENTS

1. Extract all requirements from the source (issue, spec, user request)
2. Break down into atomic, verifiable components
3. Examine the implementation (code, docs, tests)
4. Map each change to specific requirements
5. Perform gap analysis with status for each requirement:
   - Fully Satisfied / Partially Satisfied / Not Satisfied / Modified / Extra
6. Check for regressions, side effects, and standards compliance
7. Report with compliance percentages, critical gaps, and recommendations
